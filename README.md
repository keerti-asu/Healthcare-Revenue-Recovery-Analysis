# Healthcare Financial Recovery Dashboard
*An End-to-End Data Analysis Project*

## Project Overview
This project analyzes a dataset of 1,000+ healthcare claims to identify revenue leakage. I used Python to clean the data and calculate financial risk, then built a Power BI dashboard to visualize denial trends and underpayment gaps.

## Technical Skills Used
* **Python (Pandas/NumPy):** Data cleaning, handling missing values, and feature engineering (Revenue at Risk, Revenue Gap).
* **Power BI:** Data modeling, DAX measures (Denial Rate %), and interactive dashboard design.
* **Data Engineering:** ETL process to transform raw CSV data into an analysis-ready format.

## Key Insights
* **High Denial Rate:** 32.92% of billed revenue is currently at risk.
* **Root Cause:** 'Duplicate Claims' is the #1 reason for denials, suggesting a need for front-end billing audit improvements.
* **Insurance Gaps:** Commercial insurance shows the highest "Revenue Gap," indicating a need for contract renegotiation.
